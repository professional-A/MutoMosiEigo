# ホームページ設計書

**日付**: 2026-06-11  
**対象**: 武藤模試 GitHub Pages ホームページ

---

## 概要

複数の模試を科目・試験ごとに一覧できるホームページを新設する。現在の `index.html`（科学技術英語Ⅰ Unit 1&2）はサブディレクトリへ移動し、ルートに新しいホームページを置く。模試一覧は `tests.json` で管理し、新しい模試の追加はJSONの編集だけで完結する。

---

## ファイル構成

```
/
├── index.html                              ← ホームページ（新規作成）
├── tests.json                              ← 模試マニフェスト（新規作成）
└── tests/
    └── 2026-4-zenki-chukan-eigo/
        └── index.html                      ← 現在の index.html を移動
```

新しい模試を追加するときの手順：
1. `tests/` 配下に新しいフォルダを作り、模試HTMLを置く
2. `tests.json` に1エントリ追加する

---

## tests.json 仕様

```json
[
  {
    "year": 2026,
    "grade": 4,
    "exam": "前期中間試験",
    "subject": "科学技術英語Ⅰ",
    "path": "tests/2026-4-zenki-chukan-eigo/",
    "storageKey": "mutou_u12",
    "totalItems": 121
  }
]
```

| フィールド | 型 | 説明 |
|---|---|---|
| `year` | number | 年度（例: 2026） |
| `grade` | number | 学年（例: 4） |
| `exam` | string | 試験名（前期中間試験 / 前期末試験 / 後期中間試験 / 学年末試験） |
| `subject` | string | 科目名（例: 科学技術英語Ⅰ） |
| `path` | string | 模試HTMLへの相対パス（末尾スラッシュあり） |
| `storageKey` | string | localStorageのキー（模試HTML側と一致させる） |
| `totalItems` | number | 総問題数（進捗バーの分母） |

## ホームページ UI 仕様

### フィルター

ページ上部に2行のフィルターボタンを配置する。

- **科目フィルター**: 「すべて」＋ tests.json から自動生成した科目ボタン
- **試験フィルター**: 「すべて」＋「前期中間」「前期末」「後期中間」「学年末」の固定4ボタン

両フィルターは同時に適用される（AND条件）。

### 模試カード

グリッドレイアウト（1〜3列、画面幅に応じてレスポンシブ）。

各カードの表示内容：
- 年度・学年・試験名（例: `2026年度 4年 前期中間試験`）
- 科目名
- 進捗バー（回答済み問題数 / totalItems）
- 進捗テキスト（例: `42 / 69 問` または `未着手`）
- カード全体がリンク（`path` へ遷移）

進捗の取得方法：localStorageから `storageKey` のデータ（`{key: {done, correct}}` 形式）を読み、`done: true` のエントリ数を集計する。読み取れない or 0件の場合は「未着手」表示。

### グループ化

カードは「年度 × 学年」単位でグループ分けし、降順（新しい年度が上）に表示する。

```
## 2026年度 4年生
[カード] [カード] ...

## 2025年度 4年生
[カード] [カード] ...
```

フィルター適用後にグループが空になった場合は、そのグループを非表示にする。

## デザイン方針

現在の模試HTML（`index.html`）のCSS変数・フォント・カラーパレットをそのまま踏襲する。

- 背景: `--bg: #0a1626`
- アクセント: `--teal: #46d6c4`, `--amber: #f6b352`
- フォント: Outfit, Noto Sans JP, Fraunces（見出しのみ）
- 外部ライブラリ・ビルドツールなし
